# バリデーターになる

## 概要<a id="overview"></a>

:::info パーミッションレス・フェーズ1

このガイドでは、**無許可フェーズ 1**の間、既存のGCがバリデータ管理ポータルを通じてバリデ ータを管理する方法について説明する。 パーミッションレス・ネットワークを通じて誰でも参加できるようになるのは、\*\*第2段階（9月下旬予定）\*\*\*で、その時点でこのガイドは更新される。 詳細は、[パーミッションレス実装の概要](https://govforum.kaia.io/t/permissionless-implementation-overview/1218)および[PGTロードマップ](https://govforum.kaia.io/t/pgt-permissionless-governance-tokenomics-roadmap-introduction/1447)を参照のこと。

:::

カイアでは、バリデータ情報の登録・管理のために、[バリデータ管理ポータル](https://portal.kaia.io/validators)を提供しています。 このガイドでは、バリデーターがポータルを使用してカイアネットワークに参加する方法を説明します。

バリデータの操作に必要な情報はオンチェーンコントラクトに記録されていなければならないため、どのバリデータもポータルを使用して、その情報を書き込んだり更新したりするトランザクションを送信できる。 またこのポータルでは、バリデーターが新しいバリデーターをカイアネットワークにオンボーディングする際、または既存のバリデーターをカイアネットワークからオフボーディングする際に、バリデーターが所有するステーキングコントラクトを管理することができる。

ポータルは現在、以下の機能をサポートしている。

ポータルホーム](/img/nodes/become-a-validator/image01.png)

- **ホーム**: バリデータマネージャによってデプロイされたステーキングコントラクトを表示するメイン画面。
- **Deploy Staking Contract**: 新しいステーキング契約を展開します。
- **ステーキングの管理**: 配備されたステーキング契約の情報を変更します。
- **バリデーターになる**: デプロイされたステーキング契約を登録することで、バリデーターとして参加できる。
- **バリデータの管理**: オンボードバリデータの情報を変更する。
- **保留中のリクエスト**：カイアチームが使用する管理画面。

:::note

上記の機能はすべてテストネットでもサポートされている。 まずはテストネット上でテスト操作を行うことをお勧めする。

:::

## 仕事内容<a id="what-youll-do"></a>

オンボーディングの流れは、以下のページで順に説明する：

1. [前提条件](./prerequisites.md) - Validator Manager ウォレット（Kaia Safe を推奨）を接続し、必要なアカウントを準備する。
2. [Validator Onboarding](./onboarding.md) - ステーキング契約をデプロイして初期化し、オンボーディングリクエストを送信します。
3. [ステーキング契約の管理](./manage-staking.md) - 管理者の更新、KAIAのステーキング/アンステーキング、報酬アドレスの変更、マルチシグの管理など。
4. [検証者情報の管理](./manage-validator.md) - マネージャーアカウントの譲渡、オフボーディングのリクエスト、補助ステーキング契約の管理。
