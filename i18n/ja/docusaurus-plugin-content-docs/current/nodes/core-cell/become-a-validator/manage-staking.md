# ステーキング契約の管理

ステーキング管理メニュー](/img/nodes/become-a-validator/image23.png)

ステーキングの管理\*\*」メニューでは、以前に配置したステーキング契約の情報を変更することができます。

コントラクトをデプロイしたウォレットを使って接続している場合、ステーキングコントラクトのリストをブラウズして特定のコントラクトを選択し、\*\*[Manage Staking]\*\*をクリックすると、そのコントラクトの管理画面に入ることができます。

![ステーキング契約アドレスを入力](/img/nodes/become-a-validator/image24.png)

または、ステーキングコントラクトの管理ウォレットでログインした場合、ステーキングコントラクトのアドレスを \*\*[Enter a CnStaking contract address to manage it]\*\*ボックスに入力すると、管理画面に入ることができます。

ステーク概要の管理](/img/nodes/become-a-validator/image25.png)

この画面では以下のアクションを実行できる。

## 管理部門<a id="admin-management"></a>

1. **管理者の追加** - ステーキング契約の管理者を追加します。
2. **管理者の削除** - ステーキング契約管理者を削除します。
3. **更新要件** - ステーキング契約のマルチシグ閾値を変更する。

## 杭打ち作業<a id="staking-operations"></a>

![ステーク/アンステーク/出金待ち](/img/nodes/become-a-validator/image26.png)
![ステーキング操作詳細](/img/nodes/become-a-validator/image27.png)

4. **KAIAに賭ける** - 公的委任が無効になっている場合、自分の判断でKAIAに賭けることができる。
5. **アンステークKAIA** - 公開委任が無効になっている場合、ステーキング契約管理者は自分の裁量でKAIAのステーキングを解除することができる。
6. **保留中の引き出し** - アンステークボタンをクリックした後、**7日**から**14日**の間にKAIAを請求することができます。

## アドレス＆トラッカー管理<a id="address-and-tracker-management"></a>

ご褒美アドレス＆投票者](/img/nodes/become-a-validator/image28.png)

7. **報酬アドレス** - 公開委任が無効になっている場合、報酬アドレスを変更できます。
8. **Voter & Staking Tracker** - GC 投票に使用される投票者住所を変更する。 Staking Trackerが更新されると、Staking Trackerのアドレスを変更することもできます。

## 再委任<a id="redelegation"></a>

再代表](/img/nodes/become-a-validator/image29.png)

9. **再委任** - 再委任機能を有効または無効にします。この機能は、7日間のアンステーキング期間を免除し、KAIAをステーキング契約間で直ちに移動させます。 ソースとデスティネーションの両方のステーキング契約がRedelegationを有効にしていなければならない。 詳しくは[KIP-163](https://kips.kaia.io/KIPs/kip-163)を参照。

## マルチシグのリクエスト<a id="multisig-requests"></a>

マルチシグのリクエスト](/img/nodes/become-a-validator/image30.png)

10. **マルチシグ要求** - マルチシグの閾値が2以上の場合、アクション1～9のいずれかを実行するには、ステーキング契約管理者からの確認が必要です。 まだ十分な確認が集まっていないリクエストについては、各管理者がこの画面からログインし、確認フローを実行することができます。 十分な確認が集まり次第、変更要求は実行される。
