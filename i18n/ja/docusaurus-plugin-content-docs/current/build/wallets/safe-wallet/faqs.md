# よくある質問

:::caution 日没に関するお知らせ

`safe.kaia.io` は **2026年8月9日** にサービス終了となります。 今後は、[app.safe.global](https://app.safe.global) の「Safe Wallet for Kaia Network」をご利用いただき、アカウントの管理を行ってください。

:::

## Safe Globalへの移行は、既存のSafeに影響しますか？ <a id="Does moving to Safe Global affect my existing Safe"></a>

いいえ。 「Your Safe」は、オンチェーンのスマートアカウントです。 Safe Globalは、同じ契約に対する別の**UI**です。 お客様のSafeアドレス、所有者、閾値、および資産に変更はありません。 「セーフ」を再作成したり、資金を移したりする必要はありません。

詳細については、**[Safe Global への移行](./migrate-to-safe-global.md)** をご覧ください。

## 自分の「Safe」は app.safe.global に自動的に表示されますか？ <a id="Will my Safe appear automatically"></a>

ほとんどの場合、その通りです。オーナーウォレットを接続し、**Kaia** または **Kairos** を選択して、既存のセーフを開いてください。 表示されない場合は、**「既存のセーフを追加」**／\*\*「読み込み」\*\*を選択し、セーフのアドレスを貼り付けてください。

オプションのローカルデータ（アドレス帳、ニックネームなど）については、`safe.kaia.io` から一度エクスポートし、Safe Wallet の **設定 → データ** からインポートする必要がある場合があります。 これは、オンチェーン上の所有権や残高には影響しません。

## 「セーフ」を作成した後、新しい所有者を追加することはできますか？ <a id="Can i add new owners after creating a safe"></a>

はい。 Safe Walletで**設定**を開き、Safeの所有者を管理します。所有者の追加、削除、変更、または名前の変更を行うことができます。 現在の所有者としてログインしている必要があり、変更を行うには通常の承認手順が必要です。

一般的な流れ：

1. **設定** → 所有者／署名者の管理を開きます。
2. 新しい所有者（氏名＋住所）を追加してください。
3. 必要に応じて、署名ポリシーを調整してください。
4. 内容を確認して送信してください。他の所有者は、通常の「セーフ」取引と同様に確認を行います。

Safe Walletの機能改善に伴い、UIのラベルは若干変更される場合があります。最新のスクリーンショットについては、[ヘルプセンター](https://help.safe.global)をご覧ください。

## 必要な確認回数は変更できますか？ <a id="Can i change the number of required signer confirmation"></a>

はい。 **設定**で、必要な承認の閾値を変更し、その後、_現在&#x306E;_&#x30DD;リシーに従って申請を行い、所有者の署名を取得してください。

## 既存の「セーフ」を追加するにはどうすればよいですか？ <a id="How do i add an existing safe"></a>

[app.safe.global](https://app.safe.global) で、所有者のウォレットを連携するか、Safeのアドレスを追加することで、既存のSafeを開くことができます。 使用例としては、次のようなものがあります：

- 別のブラウザやデバイスから同じ「セーフ」にアクセスする
- 他の誰かがあなたを所有者に設定した金庫とのやり取り
- 読み取り専用モードで金庫を表示する

Safe Walletでは、**設定**でその機能が利用可能な場合、アドレス帳および関連データのインポート／エクスポートもサポートしています。 廃止予定の `safe.kaia.io` UI に頼るよりも、Safe Wallet で「住所」または「所有者」による接続を追加することをお勧めします。

## 一般的なセーフのセットアップに関するヒント

これといった「最適な構成」というものはありません。使用用途によって異なります。 便利なデフォルト設定：

**オーナーは何人ですか？**  
チームの場合は、複数のオーナーを設定して、複数の人物が承認できるようにしてください。 多額の残高を管理している人は、冗長性を確保するために、自身のデバイスやアカウントを複数利用することがよくあります。

**しきい値は？**  
1より大きいしきい値を設定し、1つの侵害された鍵だけでは資金を移動できないようにしてください。 所有者の約51％（例えば3人のうち2人、5人のうち3人）という閾値を設定しておくと、復旧に役立ちます。残りの所有者が、失われた所有者の役割を引き継ぐことができるからです。

**対応しているウォレットは？**   Kaia上の  
Safe Walletは、[Kaia Wallet](https://docs.kaiawallet.io/)や[MetaMask](../../tutorials/connecting-metamask.mdx)などの一般的なEOAウォレットに対応しています。 現在のウォレット一覧については、Safe Walletの接続フローをご確認ください。
